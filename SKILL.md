# Heritage News Skill

用本地 v0.3 数据库回答文化遗产新闻问题。默认中文输出；非中文来源保留原文标题和语言标记。

“本地库”指刚抓取后的 SQLite 证据缓存，不是新闻来源本身。正式推送应先刷新 verified 来源，再从缓存生成日报/周报。

微信抓取稿件只作为来源缺口和编辑判断的校准样本，不作为正式新闻来源展示。输出中只能出现一手发布机构、通讯社、地区媒体或专业机构。

这个 skill 的价值不是替代 RSS 订阅，而是把分散新闻整理成：
事件 → 议题 → 机制 → 案例比较 → 可研究问题 / 可工作线索。

## 使用方式

- 最近新闻：`最近 7 天有哪些世界遗产新闻？`
- 地区筛选：`非洲遗产地有什么新闻？`
- 主题搜索：`乌克兰遗产战争损毁情况`
- 来源筛选：`只看 UNESCO 官方消息`
- 研究线索：`最近一周有哪些值得继续研究的遗产新闻？`

## 输出格式

日报每条新闻包含：
- 中文标题；非中文来源显示“原文标题 [语言] | 中文说明”
- 来源名称 + 层级：官方 / 通讯社 / 地区 / 专业
- 发布时间
- 原文链接
- 100 字以内中文摘要
- 同一事件有多篇报道时，显示 `event_cluster_id` 和其他来源

周报或研究型问题只对高价值线索额外输出评述，不要每条新闻都硬加评论：
- 来源角色：事实来源 / 现场补充 / 机制分析线索
- 议题：这条新闻属于哪个遗产议题
- 机制：它指向什么政策、治理、保护或合作机制
- 专业框架：申遗策略、比较分析、属性识别、PNC、入遗后管理、社区参与、能力建设、遗产地生命周期、SOC / Reactive Monitoring、除名机制、融资机制等
- 比较/回溯：可联系的历史案例、同类遗产地、过往机制或横向案例
- 研究问题：可继续追问的一个问题
- 观察判断：为什么它值得进入周报，而不只是普通新闻
- 工作线索：可能连接的机构、项目、案例库或行动方向

## 抓取原则

1. 不只抓“遗产新闻”，也抓会影响遗产工作的外部机制：IUCN、自然信用、生态融资、气候、社区、旅游、灾害、土地权利等。
2. 不只看来源权威性，还要看议题是否能形成判断。输出时区分事实来源、现场补充、机制分析线索。
3. 新闻要能进入遗产专业框架。不能入框架的，只保留为普通动态，不强行拔高。
4. 做横向比较和历史回溯。单条新闻要尽量连接同类案例、既有机制和过往事件。
5. 不只生成新闻摘要，要生成观察判断：把新闻放到世界遗产机制、历史案例和现实问题里分析。

## 本地命令

```bash
python3 scripts/fetch.py --sources UNESCO,ICOMOS,Xinhua --dry-run
python3 scripts/fetch.py --all --update-db
python3 scripts/translate.py --pending
python3 scripts/deduplicate.py --update-db --days 7
python3 scripts/monitor.py --check-health
python3 scripts/digest.py --days 7 --limit 20
python3 scripts/digest.py --days 7 --limit 10 --refresh --insight
python3 scripts/radar_gap_audit.py --month 2026-06
python3 scripts/prepare.py --stats
```

## 规则

官方源优先；通讯社和地区媒体补充现场细节。第三批综合门户暂不接入。

不要只按关键词窄筛。用户问“世界遗产新闻动态”时，应包含文化遗产、自然遗产、活态遗产、博物馆治理、考古发现、遗产地管理等相关动态，并说明相关性。
